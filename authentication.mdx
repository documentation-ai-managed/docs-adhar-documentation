## Overview

Secure your Adhar documentation spaces with robust authentication and granular permissions. You manage access through user accounts, API keys, and role-based controls to ensure only authorized users view or edit content.

<Callout kind="tip">
  Always use strong passwords and rotate API keys regularly to maintain security.
</Callout>

## Account Registration and Login

Create accounts for team members to access documentation spaces.

<Steps>
  <Step title="Sign Up" icon="user-plus">
    Visit the Adhar dashboard at `https://app.adhar.dev` and click **Sign Up**.

    Enter your email and create a strong password. Verify your email to complete registration.
  </Step>
  <Step title="Log In" icon="log-in">
    Use your credentials on the login page.

    Enable two-factor authentication (2FA) in account settings for added protection.
  </Step>
  <Step title="Password Recovery" icon="key">
    Click **Forgot Password** and follow the reset link sent to your email.
  </Step>
</Steps>

<Tabs>
  <Tab title="Email/Password" icon="mail">
    Standard login for individual users.
  </Tab>
  <Tab title="SSO (Enterprise)" icon="shield">
    Configure SAML or OAuth providers in workspace settings for seamless single sign-on.
  </Tab>
</Tabs>

## API Keys and Tokens

Generate API keys to authenticate programmatic access to Adhar APIs.

<Steps>
  <Step title="Create API Key" icon="key">
    Navigate to **Account > API Keys** in the dashboard.

    Click **Generate New Key**, add a description, and copy the key immediately—it displays only once.
  </Step>
  <Step title="Use API Key" icon="code">
    Include the key in `Authorization` headers for API requests.
  </Step>
</Steps>

<CodeGroup tabs="JavaScript,Python,cURL">
  ```javascript
  const response = await fetch('https://api.adhar.dev/v1/docs', {
    headers: {
      'Authorization': `Bearer ${YOUR_API_KEY}`,
      'Content-Type': 'application/json'
    }
  });
  const data = await response.json();
  console.log(data);
  ```
  ```python
  import requests

  headers = {
      'Authorization': f'Bearer {YOUR_API_KEY}',
      'Content-Type': 'application/json'
  }
  response = requests.get('https://api.adhar.dev/v1/docs', headers=headers)
  print(response.json())
  ```
  ```bash
  curl -H "Authorization: Bearer YOUR_API_KEY" \
       -H "Content-Type: application/json" \
       https://api.adhar.dev/v1/docs
  ```
</CodeGroup>

<Request tabs="JavaScript,Python" show-lines="true">
  ```javascript
  const authResponse = await fetch('https://api.adhar.dev/v1/auth/token', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ api_key: YOUR_API_KEY })
  });
  ```
  ```python
  response = requests.post(
      'https://api.adhar.dev/v1/auth/token',
      json={'api_key': YOUR_API_KEY},
      headers={'Content-Type': 'application/json'}
  )
  ```
</Request>

<Response tabs="200,401">
  ```json
  {
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires_in": 3600,
    "token_type": "Bearer"
  }
  ```

  ```json
  {
    "error": "Invalid API key",
    "code": "unauthorized"
  }
  ```
</Response>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token obtained via API key exchange.
</ParamField>

## Role-Based Access Management

Assign roles to control permissions across documentation spaces.

<Columns cols={2}>
  <Card title="Viewer" icon="eye" href="#">
    Read-only access to published docs.
  </Card>
  <Card title="Editor" icon="edit-3" href="#">
    Edit content and manage versions.
  </Card>
  <Card title="Admin" icon="shield" href="#">
    Full control: users, spaces, billing.
  </Card>
  <Card title="Owner" icon="crown" href="#">
    Workspace ownership and deletion.
  </Card>
</Columns>

| Role   | View Docs | Edit Docs | Manage Users | Delete Space |
|--------|-----------|-----------|--------------|--------------|
| Viewer | ✅        | ❌        | ❌           | ❌           |
| Editor | ✅        | ✅        | ❌           | ❌           |
| Admin  | ✅        | ✅        | ✅           | ❌           |
| Owner  | ✅        | ✅        | ✅           | ✅           |

<Expandable title="Advanced Role Customization" default-open="false">
  Use the API to create custom roles with fine-grained permissions.

  ```javascript
  await fetch('https://api.adhar.dev/v1/roles', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${YOUR_ACCESS_TOKEN}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: 'Custom Role',
      permissions: ['read', 'write:drafts']
    })
  });
  ```
</Expandable>

<Callout kind="alert">
  Review user permissions regularly and revoke access for former team members immediately.
</Callout>
